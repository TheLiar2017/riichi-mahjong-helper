# 立直麻将助手 — 设计文档

## 1. 项目概述

**项目名称**: 立直麻将助手 (Riichi Mahjong Helper)

**核心功能**:
1. 查询日本立直麻将基础规则
2. 查询所有役种（含完整手牌示例、特殊规则、役种区分）
3. 图形化选牌计算符数与点数

**使用场景**: 线下手搓麻将时，帮助新手快速了解规则、查询役种、计算复杂符数点数。

**技术栈**: Vue 3 + Vite + TypeScript

---

## 2. 架构设计

### 目录结构

```
riichi-mahjong-helper/
├── packages/
│   ├── core/                    # 核心逻辑（可跨平台复用）
│   │   ├── src/
│   │   │   ├── data/            # 牌数据、役种数据、规则数据
│   │   │   ├── domain/          # 核心业务逻辑
│   │   │   │   ├── tiles.ts     # 牌面定义
│   │   │   │   ├── yaku.ts      # 役种定义与判定
│   │   │   │   ├── fu.ts        # 符数计算
│   │   │   │   └── score.ts     # 点数计算
│   │   │   └── index.ts         # 统一导出
│   │   └── package.json
│   └── web/                     # Vue 3 Web应用
│       ├── src/
│       │   ├── components/      # 可复用UI组件
│       │   ├── views/           # 页面视图
│       │   ├── composables/     # Vue组合式函数
│       │   ├── styles/          # 日式和风样式
│       │   └── App.vue
│       ├── index.html
│       └── package.json
```

### 设计原则

- **核心逻辑与UI分离**: `packages/core` 承载所有业务逻辑和数据，`packages/web` 仅负责展示。后续可复用 `core` 开发小程序、CLI等。
- **规则配置化**: 所有规则相关数据（役种、符数表、点 数表）以数据文件形式存在，支持不同规则变体（标准/役满倍数等）。

## 3. 视觉风格：日式和风

## 4. 功能模块

### 4.1 首页 (/)

快速入口，包含四个按钮：
- 基础规则
- 役种一览
- 符数计算器
- （预留扩展位）

### 4.2 基础规则 (/rules)

以卡片形式展示核心规则：
- 场风与自风
- 食断规则
- 振听种类（食振、立直振听、同巡振听）
- 无线电（海底捞月、岭上开花、河底捞鱼）
- 流し満貫
- 比赛规则要点

每张卡片包含规则名称、简要说明、图示。

### 4.3 役种一览 (/yaku)

页面结构：
- 顶部: 役种数量统计 + 番数筛选（一番/两番/三番/役满/累计役满）
- 主体: 役种卡片列表

役种卡片内容（详细版）：
- 役名（日文汉字 + 罗马音）
- 中文名
- 番数
- 组成原理描述
- 手牌示例（可视化牌面 + 组成图解）
- 特殊规则与注意
- 与类似役种的区分
- 常见陷阱

役种列表需覆盖约40+种标准役（视规则配置可达50+种）。

### 4.4 符数计算器 (/calculator)

用户操作流程：
1. **选牌阶段**: 图形化展示34种牌，用户点击选择手牌（最多14张，含吃/碰/杠展示）
2. **确认手牌**: 展示已选牌面，可删除或清空
3. **计算结果**:
   - 显示和牌役种（自动判定）
   - 显示符数分解（雀头、面子、搭子、特殊型）
   - 显示点数（亲家/子家分别显示）

### 4.5 预留扩展位 (/future)

为后续功能预留的空页面或路由占位。

## 5. 数据规格

### 5.1 牌面数据 (tiles)

```typescript
interface Tile {
  id: string;           // 如 "1m", "5p", "ES"（东南西北白发中）
  type: 'man' | 'pin' | 'sou' | 'honor';  // 万/筒/索/字
  value: number | string;
  name_ja: string;       // 如 "一万"
  name_zh: string;
}
```

### 5.2 役种数据 (yaku)

```typescript
interface Yaku {
  id: string;
  name_ja: string;
  name_zh: string;
  han: number;          // 番数
  han_locked?: boolean;  // 是否为役满
  description: string;  // 组成原理
  examples: HandExample[];
  special_rules?: string;
  related_yaku?: string[]; // 类似役种ID列表
  common_pitfalls?: string;
}

interface HandExample {
  tiles: string[];      // 牌面ID数组
  explanation: string;
}
```

### 5.3 符数计算规则 (fu.ts)

输入: 手牌数组 + 和牌方式（自摸/荣和）
输出: 总符数 + 符数分解说明

### 5.4 点数计算规则 (score.ts)

输入: 符数 + 番数 + 亲家/子家 + 赤宝牌
输出: 点数（支持立直棒扣除）

## 7. 状态管理

使用 Vue 3 Composition API + `provide/inject` 传递状态：
- 选牌状态（当前手牌、已吃碰杠的牌）
- 役种筛选状态
- 规则配置（当前使用哪套规则变体）

无需引入Vuex/Pinia，保持轻量。

---

## 8. 性能考量

- 役种数据按需加载，非全量预加载
- 牌面图片使用懒加载
- 计算器逻辑在 `packages/core` 中，无UI依赖，可快速响应

---

## 9. 开发优先级

1. **Phase 1**: 项目脚手架（packages/core + packages/web），核心数据定义
2. **Phase 2**: 规则页面 + 役种列表页面
3. **Phase 3**: 符数计算器（选牌UI + 计算逻辑）
4. **Phase 4**: 完善役种详情（示例图解）
5. **Phase 5**: PWA支持（可选）
